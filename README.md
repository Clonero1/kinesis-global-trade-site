# Сайт Kinesis Global Trade

Проект B2B/B2C-сайта для ООО «Кинезис Глобал Трейд».

Цель проекта - создать сайт-каталог для продажи запчастей для карьерной, горнодобывающей, строительной и спецтехники с возможностью дальнейшего развития до маркетплейса.

## Зафиксированный стек

- Next.js;
- TypeScript;
- PostgreSQL;
- Prisma;
- Redis/BullMQ в будущем для очередей и фоновых задач;
- S3-совместимое хранилище в будущем для файлов, фото, документов и XML-выгрузок.

## Роль сайта

Сайт является витриной, каталогом, поиском по артикулу, массовым поиском, личным кабинетом и интеграционным слоем между клиентом, 1С, Битрикс24, Авито и службами доставки.

Главным источником товаров, цен, остатков, складов и документов является 1С. Сайт хранит локальную синхронизированную копию данных из 1С.

Заявка всегда сначала сохраняется в базе сайта, затем передается в 1С через интеграционный слой. Если 1С недоступна, заявка не должна теряться.

## Текущий статус

Сейчас подготовлена техническая основа проекта:

- Prisma-схема базы данных;
- базовые конфиги Next.js и TypeScript;
- централизованное чтение переменных окружения;
- singleton Prisma Client;
- чистые функции нормализации артикула;
- структура будущих модулей;
- документация по локальному запуску и структуре проекта.

Пользовательский интерфейс, страницы каталога, API бизнес-логики, авторизация, реальное подключение к 1С и внешним сервисам пока не реализуются.

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Скопировать переменные окружения:

```bash
cp .env.example .env
```

На Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

3. Заполнить `DATABASE_URL` в `.env`.

4. Сгенерировать Prisma Client:

```bash
npm run prisma:generate
```

5. Запустить миграции, когда они будут подготовлены:

```bash
npm run prisma:migrate
```

6. Запустить dev-сервер:

```bash
npm run dev
```

На этом этапе страницы сайта намеренно не созданы, потому что текущая задача - только технический каркас.
