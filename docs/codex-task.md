# Задача для Codex

## Контекст

Есть проект сайта Kinesis Global Trade. Это не просто корпоративный сайт, а будущий рабочий инструмент продаж запчастей для карьерной, горнодобывающей, строительной и спецтехники.

Сайт должен начинаться как B2B/B2C-каталог с заявками, но архитектура должна позволять развить его до маркетплейса.

## Что нужно сделать сначала

Не писать код сразу.

Сначала нужно изучить документы проекта и подготовить архитектурное предложение.

## Основные документы

- `README.md`
- `docs/business-requirements.md`
- `docs/technical-specification.md`
- `docs/integrations.md`
- `docs/product-import-format.md`
- `docs/questions-for-developers.md`

## Задача

Проанализируй документы и подготовь предложение по архитектуре сайта.

Нужно описать:

1. Рекомендуемую платформу или стек разработки.
2. Структуру модулей сайта.
3. Структуру базы данных.
4. Логику каталога товаров.
5. Логику поиска по артикулу.
6. Логику массового поиска товаров.
7. Логику импорта из Excel, CSV, XML и 1С.
8. Логику личного кабинета покупателя.
9. Логику оформления заявки или заказа.
10. Логику интеграции с Битрикс24.
11. Логику интеграции с Авито.
12. Логику расчета доставки через СДЭК и Деловые Линии.
13. Как в будущем расширить проект до маркетплейса.
14. Основные риски проекта.
15. Вопросы, которые нужно согласовать до начала разработки.

## Важные ограничения

1. Артикул — главный идентификатор товара.
2. Нельзя автоматически считать товары аналогами только по названию.
3. Состояние товара должно храниться отдельно: новый, б/у, восстановленный, под заказ.
4. Склад должен отображаться в карточке и поиске.
5. Массовый поиск должен быть в первом этапе.
6. Проект должен быть масштабируемым.

## Ожидаемый результат

Подготовить документ `docs/architecture-proposal.md`.

В документе нужно дать не общие слова, а конкретную архитектуру с таблицами, модулями, потоками данных и рисками.
