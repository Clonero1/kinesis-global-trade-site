# Зафиксированные архитектурные решения

## 1. Технологический стек

Проект разрабатывается на следующем стеке:

- Next.js;
- TypeScript;
- PostgreSQL;
- Prisma;
- Redis/BullMQ для фоновых задач и очередей;
- S3-совместимое хранилище для файлов, фото, документов и XML-выгрузок.

1С-Битрикс как основная платформа проекта не используется.

## 2. Источник товарных данных

Главным источником товаров, цен, остатков и складской информации является 1С.

Сайт не должен становиться главным местом хранения и редактирования товарной истины. Его задача — получать данные из 1С, отображать их в каталоге, обеспечивать поиск, массовый поиск, заявки и пользовательские сценарии.

## 3. Документы и заказы

Формирование заказов, счетов, договоров и других официальных документов выполняется в 1С.

Сайт должен передавать в 1С данные заявки или заказа, а затем получать обратно статусы и ссылки/файлы документов, если это будет реализовано технически.

## 4. Роль сайта

Сайт является:

- публичной витриной каталога;
- инструментом поиска по артикулу;
- инструментом массового поиска;
- точкой приема заявок;
- личным кабинетом клиента;
- интерфейсом для взаимодействия с каталогом и заказами;
- промежуточным слоем между клиентом, 1С, CRM и внешними интеграциями.

Сайт не является главным источником бухгалтерских, складских и юридически значимых документов.

## 5. Следствие для архитектуры

Разработка должна строиться вокруг интеграции с 1С:

1. Сначала сайт сохраняет заявку у себя.
2. Затем передает заявку в 1С через интеграционный слой.
3. 1С формирует заказ и документы.
4. Сайт получает статус обработки и документы/ссылки на документы.
5. Если обмен с 1С временно недоступен, заявка не должна теряться: она остается на сайте и повторно отправляется через очередь.

## 6. Критичный риск

Главный риск проекта — не сам сайт, а качество и доступность обмена с 1С.

До начала активной разработки нужно отдельно согласовать:

- формат обмена с 1С;
- периодичность выгрузки товаров, цен и остатков;
- способ передачи заявок/заказов в 1С;
- кто дорабатывает 1С;
- кто отвечает за качество номенклатуры, артикулов и фото в 1С;
- где именно будут редактироваться карточки товаров.
