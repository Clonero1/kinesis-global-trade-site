# Интеграции проекта

## 1. Интеграция с 1С

1С должна быть основным источником данных по товарам, ценам, остаткам и заказам, если это возможно технически.

### Данные из 1С на сайт

- номенклатура;
- артикулы;
- дополнительные каталожные номера;
- производитель;
- наименование;
- описание;
- цены;
- остатки;
- склады;
- состояние товара;
- фото, если они есть в 1С;
- признаки активности товара.

### Данные с сайта в 1С

- заявки;
- заказы;
- данные клиента;
- реквизиты юридического лица;
- состав заказа;
- выбранный способ доставки;
- комментарии клиента;
- источник обращения.

## 2. Импорт Excel / CSV

Excel или CSV нужны как резервный и промежуточный способ загрузки товаров.

### Когда нужен Excel

- если прямой обмен с 1С еще не готов;
- если партнер присылает прайс-лист;
- если нужно быстро загрузить отдельную группу товаров;
- если нужно проверить или обогатить данные перед загрузкой.

### Требования

1. Должен быть понятный шаблон загрузки.
2. Должна быть проверка ошибок.
3. Должен быть отчет по загруженным, обновленным и отклоненным позициям.
4. Должна быть защита от дублей.
5. Сопоставление должно идти по артикулу, каталожному номеру или внутреннему коду.

## 3. XML

XML может использоваться для:

- обмена с сайтом;
- выгрузки на Авито;
- обмена с внешними площадками;
- передачи данных партнерам.

## 4. Авито

Сайт должен учитывать связь с Авито как с текущим каналом продаж.

### Возможные функции

1. Формирование XML-файла для Авито.
2. Контроль товаров, которые есть на сайте, но отсутствуют в XML.
3. Контроль товаров, которые есть в XML, но отсутствуют в текущих остатках.
4. Обновление цен и остатков.
5. Снятие с публикации отсутствующих товаров.
6. Подготовка фото и описаний для объявлений.

## 5. СДЭК

Интеграция с СДЭК нужна для расчета доставки и, возможно, создания отправлений.

### Функции

- расчет стоимости доставки;
- расчет срока доставки;
- выбор пункта выдачи;
- адресная доставка;
- передача данных доставки в заказ.

## 6. Деловые Линии

Интеграция с Деловыми Линиями нужна для крупногабаритных и тяжелых запчастей.

### Функции

- расчет стоимости перевозки;
- расчет срока доставки;
- выбор терминала;
- учет веса и габаритов;
- передача данных доставки в заказ.

## 7. Битрикс24

Битрикс24 может использоваться как CRM для обработки заявок.

### Возможные сценарии

1. Заявка с сайта создает лид в Битрикс24.
2. В лид передаются контактные данные, список товаров и комментарий клиента.
3. Менеджер обрабатывает заявку в CRM.
4. Источник обращения фиксируется как сайт.

## 8. Основные вопросы перед разработкой

1. Что является главным источником товаров: 1С, сайт или отдельная PIM-система?
2. Какой формат обмена с 1С доступен сейчас?
3. Можно ли автоматически передавать заказы с сайта в 1С?
4. Где будут храниться фото товаров?
5. Где будут редактироваться карточки товаров?
6. Кто отвечает за качество данных по артикулам и описаниям?
7. Как часто должны обновляться цены и остатки?
