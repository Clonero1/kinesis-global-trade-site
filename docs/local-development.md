# Локальная разработка

## 1. Установить зависимости

```bash
npm install
```

## 2. Скопировать переменные окружения

```bash
cp .env.example .env
```

На Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

## 3. Заполнить DATABASE_URL

В `.env` нужно указать строку подключения к PostgreSQL:

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/kinesis_global_trade?schema=public"
```

`DATABASE_URL` является обязательной переменной для работы с базой данных и Prisma.

Остальные переменные внешних интеграций на текущем этапе могут оставаться заглушками.

## 4. Сгенерировать Prisma Client

```bash
npm run prisma:generate
```

## 5. Запустить миграции

Когда миграции будут подготовлены, выполнить:

```bash
npm run prisma:migrate
```

Сейчас схема Prisma уже описана, но миграционные файлы еще не создавались.

## 6. Запустить dev-сервер

```bash
npm run dev
```

На текущем этапе пользовательские страницы намеренно не создаются. Каркас нужен для дальнейшей безопасной разработки каталога, поиска, массового поиска, заявок и интеграции с 1С.

## Что не подключается на этом этапе

На этом этапе не подключаются:

- реальная 1С;
- Битрикс24;
- СДЭК;
- Деловые Линии;
- Авито;
- S3-хранилище;
- онлайн-оплата.

Также не создаются:

- страницы каталога;
- пользовательский интерфейс;
- API routes;
- авторизация;
- XML-выгрузка Авито;
- тестовые товары;
- кабинет продавца;
- полноценный маркетплейс;
- автоматическое формирование документов на сайте.
