# Формат импорта товаров

## 1. Назначение

Документ описывает требования к загрузке товаров, цен и остатков на сайт.

Импорт должен поддерживать загрузку из 1С, Excel, CSV или XML. На первом этапе допустима загрузка через Excel, но архитектура должна учитывать дальнейшую автоматизацию.

## 2. Главный принцип сопоставления

Главный идентификатор товара — артикул или каталожный номер.

Название товара не является надежным идентификатором. Одинаковые или похожие названия не означают, что товары взаимозаменяемы.

## 3. Минимальные поля товара

| Поле | Обязательность | Описание |
|---|---:|---|
| internal_id | желательно | Внутренний код товара из 1С или другой системы |
| article | обязательно | Основной артикул товара |
| additional_articles | желательно | Дополнительные каталожные номера через разделитель |
| name | обязательно | Наименование товара |
| brand | желательно | Производитель |
| category | желательно | Категория товара |
| condition | обязательно | Новый, б/у, восстановленный, под заказ |
| warehouse | желательно | Склад |
| stock_qty | обязательно | Остаток |
| price | желательно | Цена |
| currency | желательно | Валюта |
| description | желательно | Описание |
| photo_url | желательно | Ссылка на фото |
| weight | желательно | Вес |
| length | желательно | Длина |
| width | желательно | Ширина |
| height | желательно | Высота |
| is_active | обязательно | Показывать товар на сайте или нет |

## 4. Требования к артикулам

Перед сопоставлением артикулы нужно нормализовать:

1. Удалять лишние пробелы.
2. Приводить к единому регистру.
3. Учитывать варианты с дефисом и без дефиса.
4. Не объединять товары автоматически без проверки, если совпадение не точное.
5. Хранить исходное написание артикула, которое пришло от поставщика или из 1С.

## 5. Состояние товара

Состояние товара должно быть отдельным полем, а не частью названия.

Возможные значения:

- new — новый;
- used — б/у;
- remanufactured — восстановленный;
- repair_after — после ремонта;
- on_order — под заказ.

## 6. Склады

Склад влияет на интерпретацию состояния товара. Например, восстановленные форсунки, б/у детали и новые детали могут находиться на разных складах.

В карточке товара и в результатах поиска нужно показывать склад и состояние позиции.

## 7. Результат импорта

После загрузки система должна показать отчет:

- всего строк в файле;
- успешно загружено;
- обновлено существующих товаров;
- создано новых товаров;
- пропущено;
- ошибки;
- дубли;
- строки, требующие ручной проверки.

## 8. Ошибки импорта

Ошибки должны быть понятны пользователю.

Примеры:

- не указан артикул;
- не указано название;
- некорректный остаток;
- некорректная цена;
- найден дубль артикула;
- не определено состояние товара;
- не определена категория.

## 9. Массовый поиск по импортированным данным

Массовый поиск должен работать по тем же правилам сопоставления, что и импорт.

Пользователь загружает или вставляет список артикулов. Система возвращает таблицу со статусом по каждой строке.

Статусы:

- найдено в наличии;
- найдено, но нет остатка;
- доступно под заказ;
- не найдено;
- требуется ручная проверка.
