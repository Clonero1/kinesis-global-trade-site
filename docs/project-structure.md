# Структура проекта

## prisma/

Папка схемы базы данных Prisma.

Содержит `schema.prisma`, где описаны модели PostgreSQL: каталог, артикулы, цены, остатки, склады, заявки, интеграционные задачи, логи, документы и минимальный задел под продавцов.

## src/lib/

Общие технические утилиты проекта.

Сейчас содержит:

- `prisma.ts` - безопасный singleton Prisma Client;
- `env.ts` - централизованное чтение переменных окружения;
- `article-normalization.ts` - чистые функции нормализации артикула.

## src/modules/

Папка будущих предметных модулей.

Модули пока содержат только README с границами ответственности. Бизнес-логика, API и UI на этом этапе не реализуются.

## src/modules/catalog/

Будущий модуль каталога.

Отвечает за товары, бренды, категории, состояния, артикулы, изображения, характеристики, цены, остатки и склады.

Главный источник товарных данных - 1С.

## src/modules/search/

Будущий модуль точного поиска по артикулу.

Поиск должен строиться по `ProductIdentifier.normalizedValue`. Похожее название товара не является доказательством аналога.

## src/modules/bulk-search/

Будущий модуль массового поиска товаров по списку артикулов.

Каждая входная строка должна получать отдельный результат: найдено, не найдено, несколько вариантов или требуется проверка.

## src/modules/requests/

Будущий модуль заявок.

Заявка всегда сначала сохраняется в базе сайта. Строки заявки должны хранить снимки цены, остатка, склада, названия и состояния товара.

## src/modules/integrations/

Будущий общий модуль интеграций.

Отвечает за интеграционные задачи, логи, повторные попытки, идемпотентность и изоляцию внешних систем от бизнес-модулей.

## src/modules/integrations/onec/

Будущий модуль обмена с 1С.

1С является главным источником товаров, цен, остатков, складов и документов. Реальное подключение на текущем этапе не реализуется.

## src/modules/admin/

Будущий модуль базовой админки.

Должен показывать товары, импорты, заявки, статусы отправки в 1С и логи ошибок. Не должен заменять 1С как источник товарной истины.

## src/modules/documents/

Будущий модуль документов.

Сайт не формирует официальные документы. Он получает документы или ссылки на документы из 1С и связывает их с заявками.

## src/modules/delivery/

Будущий модуль доставки.

Должен хранить выбранную доставку и в будущем поддержать СДЭК и Деловые Линии. Реальные интеграции на текущем этапе не подключаются.

## docs/

Папка проектной документации.

Содержит архитектурные решения, описание базы, решения по базе, интеграционное ТЗ по 1С, дорожную карту, состав первого рабочего релиза и инструкции для разработки.
