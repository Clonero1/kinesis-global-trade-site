# Поэтапное техническое задание

## Этап 1. Базовый сайт-каталог и прием заявок

Цель этапа — быстро запустить рабочую версию сайта, которая уже сможет принимать заявки и показывать товары.

### Функции

1. Главная страница.
2. Страница о компании.
3. Каталог товаров.
4. Категории товаров.
5. Карточка товара.
6. Поиск по артикулу.
7. Массовый поиск товаров по списку артикулов.
8. Форма заявки.
9. Корзина или список выбранных позиций для отправки менеджеру.
10. Страница контактов.
11. Базовая административная панель.

### Карточка товара

В карточке товара должны быть:

- название;
- основной артикул;
- дополнительные каталожные номера;
- производитель;
- применимость, если известна;
- состояние товара;
- цена;
- остаток;
- склад;
- фото;
- описание;
- кнопка отправки заявки.

### Массовый поиск

Пользователь должен иметь возможность вставить список артикулов в текстовое поле или загрузить файл. Сайт должен вернуть таблицу результатов.

Результат массового поиска:

- исходный артикул пользователя;
- найденный артикул;
- название товара;
- цена;
- остаток;
- склад;
- состояние;
- статус: найдено, нет в наличии, не найдено;
- возможность добавить найденные позиции в заявку.

## Этап 2. Интеграция с 1С и автоматическое обновление данных

Цель этапа — убрать ручное обновление каталога и остатков.

### Функции

1. Импорт товаров из 1С.
2. Импорт остатков.
3. Импорт цен.
4. Обновление существующих карточек по артикулу или внутреннему коду.
5. Лог ошибок импорта.
6. Возможность ручной проверки спорных позиций.
7. Настройка расписания обмена.

### Варианты обмена

- прямой обмен с 1С;
- загрузка Excel;
- загрузка XML;
- загрузка CSV;
- промежуточный API.

Финальный способ обмена нужно согласовать с IT-директором и разработчиками.

## Этап 3. Личный кабинет покупателя

Цель этапа — дать клиентам возможность работать с заказами системно.

### Функции

1. Регистрация и авторизация.
2. Профиль физического лица.
3. Профиль юридического лица.
4. Реквизиты организации.
5. История заявок и заказов.
6. Повтор заказа.
7. Сохраненные списки артикулов.
8. Документы по заказу.
9. Статусы заказов.

## Этап 4. Автоматическое формирование документов

Цель этапа — ускорить работу с юридическими лицами.

### Функции

1. Формирование счета.
2. Формирование договора или счет-договора.
3. Передача заказа в 1С.
4. Получение статуса оплаты.
5. Хранение документов в личном кабинете.

## Этап 5. Расчет доставки

Цель этапа — дать клиенту понимание стоимости и вариантов доставки.

### Интеграции

- СДЭК;
- Деловые Линии;
- другие транспортные компании при необходимости.

### Функции

1. Выбор города доставки.
2. Расчет стоимости доставки.
3. Расчет срока доставки.
4. Выбор пункта выдачи или адресной доставки.
5. Передача данных доставки в заказ.

## Этап 6. SEO и продвижение

Цель этапа — обеспечить индексацию товаров и привлечение поискового трафика.

### Функции

1. Человекопонятные адреса страниц.
2. Title и description для страниц.
3. Sitemap.xml.
4. Robots.txt.
5. Микроразметка товара.
6. Индексация карточек товаров и категорий.
7. Страницы под группы товаров и бренды.
8. Контроль дублей карточек.

## Этап 7. Масштабирование до маркетплейса

Цель этапа — подготовить сайт к подключению сторонних продавцов.

### Функции

1. Личный кабинет продавца.
2. Загрузка товаров продавцом.
3. Модерация карточек.
4. Разделение остатков и цен по продавцам.
5. Заказы по продавцам.
6. Комиссионная модель или другой способ монетизации.
7. Контроль качества данных.
8. Отчеты по продажам продавцов.
