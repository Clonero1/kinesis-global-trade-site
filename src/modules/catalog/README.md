# Catalog module

## Назначение

Модуль каталога отвечает за будущую работу с локальной копией товаров, брендов, категорий, артикулов, изображений, характеристик, цен, остатков и складов.

Основной источник товарных данных - 1С. Сайт хранит синхронизированную копию для отображения и поиска.

## Что модуль должен делать в будущем

- читать товары из `Product`;
- показывать основной и дополнительные артикулы из `ProductIdentifier`;
- читать цены из `ProductPrice`;
- читать остатки из `ProductStock`;
- связывать остатки со складами из `Warehouse`;
- учитывать состояние товара через `ProductCondition`;
- поддерживать данные, пришедшие из 1С.

## Что модуль пока не делает

- не создает пользовательский интерфейс каталога;
- не создает страницы Next.js;
- не реализует API;
- не редактирует товарную истину вместо 1С;
- не хранит цену и остаток как главные поля `Product`;
- не считает похожие товары аналогами по названию.

## Важные правила

`Product` не должен хранить цену и остаток как главные поля. Цены находятся в `ProductPrice`, остатки находятся в `ProductStock`, артикулы находятся в `ProductIdentifier`.
