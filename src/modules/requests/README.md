# Requests module

## Назначение

Модуль заявок отвечает за будущий прием заявки или корзины покупателя.

## Что модуль должен делать в будущем

- сохранять заявку в `Request`;
- сохранять строки заявки в `RequestItem`;
- сохранять доставку в `DeliveryInfo`;
- связывать заявку с `Customer` и `CustomerCompany`;
- создавать отдельную интеграционную задачу для отправки в 1С;
- хранить статус отправки в 1С.

## Что модуль пока не делает

- не создает форму заявки;
- не создает корзину;
- не создает API;
- не отправляет данные в 1С;
- не формирует документы;
- не подключает онлайн-оплату.

## Важные правила

Заявка всегда сохраняется локально. `RequestItem` хранит снимки цены, остатка, склада, названия и состояния товара на момент отправки.

Отправка в 1С идет отдельной интеграционной задачей. Недоступность 1С не должна приводить к потере заявки.
